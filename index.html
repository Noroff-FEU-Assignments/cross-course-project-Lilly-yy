Web Security

How would you advise a potential client to secure his/her website from web security attacks? Note: the word count for your answer should not be greater than 250 words

I would advise the client to follow these best practices in web security:

1. Sanitization of data.

They should either manually or by using a well-known third-party solution make sure the data they receive on the website is clean and in the correct format.

2. Secure authorization and authentication.

They should take measures to ensure that only authorized users can access what they are permitted to access. OneLogin and Okta could be helpful tools for this.

3. Using trustworthy third-party apps.

Before starting to use a third-party application, I would highly suggest for the client to research the apps. Cheap/free is very often not good. A look at how many times per week the app is downloaded would be recommended.

4. Continuous Deployment/Continuous Integration.

I would recommend the client to set up automatic processes to check, test and deploy code.

5. Network security.

The client should also make sure to protect the data that moves between computers and servers.

6. Automatic user timeout.

Lastly I would advise that the client sets up automatic user timeout so that the user gets kicked out of the website if they have been inactive for a certain amount of time. This is a measure that can prevent wrong users from getting access if the right user forgets to log out.
